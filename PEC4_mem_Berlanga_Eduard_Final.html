1. INTRODUCCIÓN Y OBJETIVOS

El presente documento recoge la memoria final del proyecto RunTogether, una plataforma web integral diseñada para la gestión y participación en eventos deportivos de running. Este proyecto ha sido desarrollado a lo largo del semestre a través de entregas incrementales (PEC 2, PEC 3 y PEC 4), culminando en una aplicación Full Stack completamente funcional y desplegada en la nube.

El objetivo principal ha sido crear una solución que permita a los organizadores publicar carreras y a los corredores inscribirse, interactuar y formar comunidad, aplicando las mejores prácticas de desarrollo web moderno.

2. EVOLUCIÓN DEL PROYECTO

2.1. PEC 2: Fundamentos del Backend

En esta primera fase de desarrollo, se construyó la lógica del servidor. Se implementó una API RESTful utilizando Node.js y Express. Los hitos clave fueron:

  • Diseño del modelo de datos con Prisma ORM (Usuarios, Eventos, Inscripciones).
  • Implementación de autenticación segura con JWT y hashing de contraseñas con Bcrypt.
  • Creación de endpoints CRUD básicos para eventos y usuarios.
  • Validación de datos con express-validator.

2.2. PEC 3: Desarrollo del Frontend

La segunda fase se centró en la interfaz de usuario. Se desarrolló una Single Page Application (SPA) con React y Vite. Los logros principales incluyeron:

  • Diseño responsive y moderno utilizando Tailwind CSS.
  • Gestión de rutas con react-router-dom (Públicas y Privadas).
  • Consumo de la API mediante Axios e interceptores para el manejo de tokens.
  • Contexto global de autenticación (AuthContext) para gestionar la sesión del usuario.
  • Vistas principales: Home, Login, Registro, Detalle de Evento y Perfil.

2.3. PEC 4: Integración Final y Mejoras

En esta fase final, se ha pulido la aplicación y se han añadido funcionalidades avanzadas solicitadas, además de asegurar el despliegue. Las novedades incluyen:

Novedades Destacadas:
  • Sistema de Notificaciones: Alertas en tiempo real para inscripciones y actividad social.
  • Panel de Administración Completo: Gestión integral de usuarios y eventos (Crear, Editar, Eliminar) desde el frontend.
  • Muro Social y Comentarios: Interacción comunitaria completa.
  • Lista de Participantes: Visualización pública de inscritos en cada evento.
  • Despliegue en Producción: Backend en Render y Frontend en Netlify.

Base de Datos MongoDB + Prisma Persistencia de datos y ORM.

3.2. Estructura de Base de Datos

El esquema relacional definido en Prisma incluye los siguientes modelos principales:

  • User: Almacena credenciales, perfil y rol (ADMIN/RUNNER).
  • Event: Datos de la carrera (fecha, lugar, precio, dificultad).
  • Inscription: Tabla intermedia para relacionar Usuarios con Eventos.
  • Post/Comment: Elementos sociales para el muro y comentarios de eventos.
  • Notification: Alertas del sistema para el usuario.

4. FUNCIONALIDADES IMPLEMENTADAS

4.1. Gestión de Usuarios

El sistema permite el ciclo completo de vida del usuario:

  • Registro: Validación de campos, email único y aceptación de términos (checkbox implementado).
  • Login: Generación de Token JWT persistente.
  • Perfil: Edición de datos personales, avatar y cambio de contraseña.

4.2. Gestión de Eventos

El núcleo de la aplicación. Los usuarios pueden:

  • Ver listado de eventos con filtros (fecha, dificultad, ubicación).
  • Ver detalle del evento: mapa, descripción, precio y lista de participantes.
  • Inscribirse (con validación de plazas máximas) y cancelar inscripción.

4.3. Comunidad

Se ha implementado un componente social robusto:

  • Muro Social: Los usuarios publican experiencias y fotos.
  • Interacciones: Sistema de "Likes" y comentarios en posts.
  • Comentarios en Eventos: Sección de preguntas y respuestas en cada evento.

4.4. Sistema de Notificaciones

Se ha desarrollado un sistema de notificaciones que alerta al usuario en la barra de navegación (icono de campana con contador) cuando:

  • Se inscribe exitosamente a un evento.
  • Alguien comenta en su publicación.
  • Alguien da "like" a su publicación.

4.5. Panel de Administración

Acceso exclusivo para usuarios con rol ADMIN. Permite:

  • Ver estadísticas globales (total usuarios, eventos, inscripciones).
  • Gestión de Usuarios: Ver lista, cambiar roles, eliminar usuarios.
  • Gestión de Eventos: Formulario completo para Crear, Editar y Eliminar eventos sin tocar código.

5. MANUAL DE INSTALACIÓN Y DESPLIEGUE

5.1. Instalación Local

Para ejecutar el proyecto en local, siga estos pasos:

Backend

cd backend npm install # Crear archivo .env basado en .env.example npx prisma migrate dev --name init npm run seed # Carga usuarios de prueba npm run dev

Frontend

cd frontend npm install # Crear archivo .env con VITE_API_URL=http://localhost:3000/api npm run dev

Usuarios de Prueba:

  • Admin: admin@runtogether.com / admin123
  • Runner: runner@test.com / runner123

5.2. Despliegue en la Nube

La aplicación se encuentra operativa en las siguientes URLs:

  • Frontend: https://runthoger2020.netlify.app
  • Backend: https://runtogether.onrender.com

6. CONCLUSIONES

El desarrollo de RunTogether ha permitido consolidar conocimientos avanzados de desarrollo web Full Stack. Se ha logrado una aplicación robusta, segura y escalable.

La transición de una API básica en la PEC 2 a una aplicación completa con interfaz reactiva en la PEC 3, y finalmente la adición de características complejas como notificaciones y administración en la PEC 4, demuestra la capacidad de evolución del software.

Como líneas de trabajo futuro, se podría implementar un chat en tiempo real mediante WebSockets, integración con pasarelas de pago reales (Stripe) y una aplicación móvil nativa utilizando React Native.
