1. INTRODUCCIÓN

1.1. Contexto y justificación del Trabajo

RunTogether es una plataforma web diseñada para conectar organizadores de eventos de running con corredores interesados en participar. El proyecto surge de la necesidad de facilitar la organización y descubrimiento de eventos deportivos, proporcionando una solución tecnológica moderna y eficiente.

En la PEC 1 se realizó el análisis completo del proyecto, definiendo requisitos funcionales y no funcionales. Esta memoria (PEC 2) justifica la implementación técnica del backend, pieza fundamental para la lógica de negocio y persistencia de datos.

1.2. Objetivos del Trabajo

Objetivo General: Desarrollar el backend completo de la plataforma RunTogether, implementando una API REST funcional con Node.js y Express, conectada a una base de datos MongoDB mediante Prisma ORM.

Objetivos Específicos

  • OE1 - Configuración: Configurar el entorno de desarrollo backend con Node.js y las dependencias necesarias.
  • OE2 - Datos: Diseñar e implementar el modelo de datos con Prisma y MongoDB.
  • OE3 - Seguridad: Desarrollar un sistema de autenticación seguro basado en JWT y bcrypt.
  • OE4 - Funcionalidad: Implementar endpoints para la gestión completa de eventos e inscripciones.
  • OE5 - Calidad: Asegurar la calidad mediante validaciones exhaustivas y pruebas funcionales.

1.3. Impacto en sostenibilidad, ético-social y de diversidad

Impacto Social y Diversidad

El proyecto fomenta la actividad física y la salud (ODS 3: Salud y Bienestar). La plataforma incluye funcionalidades inclusivas como "Grupos de Ritmo" para corredores de todos los niveles, desde principiantes hasta expertos, promoviendo la diversidad en el deporte y evitando la exclusión por rendimiento físico.

Impacto Medioambiental

Tecnológicamente, se ha optado por una arquitectura eficiente (Node.js) que maximiza el rendimiento con menor consumo de recursos de servidor. El uso de MongoDB permite un almacenamiento flexible sin redundancia excesiva de datos, contribuyendo a reducir la huella de carbono digital del almacenamiento en la nube.

1.4. Enfoque y método seguido

Se ha seguido una metodología ágil (Scrum) adaptada al desarrollo académico. La estrategia técnica se basa en una arquitectura MVC (Modelo-Vista-Controlador) para separar responsabilidades, utilizando tecnologías modernas (Node.js, Prisma) que permiten un desarrollo rápido y escalable.

2.2. Arquitectura del Backend

Se ha implementado una arquitectura basada en el patrón MVC (Modelo-Vista-Controlador), lo que garantiza una clara separación de responsabilidades y facilita el mantenimiento del código.

Figura 1: Arquitectura MVC implementada en el proyecto RunTogether

  • Modelos (Prisma): Definen la estructura de datos y las relaciones.
  • Controladores: Contienen la lógica de negocio y procesan las peticiones.
  • Rutas: Gestionan los endpoints y aplican middleware de seguridad.

Estructura de Carpetas

backend/ ├── src/ │ ├── controllers/ # Lógica de negocio (auth, event, registration) │ ├── routes/ # Definición de endpoints │ ├── middleware/ # Autenticación y validación │ └── config/ # Configuración de Prisma y variables ├── prisma/ │ └── schema.prisma # Definición del modelo de datos └── server.js # Punto de entrada de la aplicación

2.3. Modelado de Datos

El diseño de la base de datos es crucial para el funcionamiento de la plataforma. Se han definido tres modelos principales en schema.prisma que interactúan entre sí.

Figura 2: Diagrama Entidad-Relación simplificado

Modelo de Evento (Prisma Schema)

model Event { id String @id @default(auto()) @map("_id") @db.ObjectId title String description String organizerId String @db.ObjectId organizer User @relation("EventOrganizer", fields: [organizerId], references: [id]) eventType EventType date DateTime location Json // { city, address, coordinates } distance Float? difficulty Difficulty @default(MEDIUM) price Float @default(0) // FUNCIONALIDADES DIFERENCIALES terrain Json? // { type, elevation, technicalDifficulty } paceGroups Json[] // [{ name, targetPace, maxParticipants }] registrations Registration[] createdAt DateTime @default(now()) updatedAt DateTime @updatedAt }

2.4. Sistema de Autenticación

Implementado con JWT (JSON Web Tokens) y bcrypt para el hashing de contraseñas. El flujo incluye registro, login, y middleware de protección de rutas.

  • Registro: Valida email único y encripta contraseña (12 rounds).
  • Login: Verifica credenciales y emite token con validez de 7 días.
  • Middleware: Intercepta peticiones, verifica la firma del token y adjunta el usuario a la request.

2.5. Validación y Seguridad

Se utiliza express-validator para sanitizar y validar entradas, asegurando que los datos recibidos cumplen con los formatos esperados antes de ser procesados.

Adicionalmente, se han implementado medidas de seguridad como:

  • Helmet: Configuración de headers HTTP seguros.
  • CORS: Control de acceso a recursos desde dominios externos.
  • Rate Limiting: (Preparado para futura implementación) para prevenir ataques DDoS.

2.6. Funcionalidades Diferenciales

Se han añadido características avanzadas que distinguen a esta plataforma:

  • Grupos de Ritmo: Permite a los organizadores definir subgrupos dentro de un evento (ej. "Sub 45min", "Sub 50min"), mejorando la experiencia del corredor.
  • Información de Terreno: Estructura JSON compleja para detallar desnivel, tipo de superficie y dificultad técnica.
  • Búsqueda Avanzada: Endpoint específico que permite filtrar por múltiples criterios simultáneos (ciudad, fecha, distancia, dificultad).

5. GLOSARIO

API REST
Interfaz de programación de aplicaciones que cumple con los principios de diseño REST (Representational State Transfer).

JWT (JSON Web Token)
Estándar abierto para la creación de tokens de acceso que permiten la propagación de identidad y privilegios.

Middleware
Software que actúa como puente entre un sistema operativo o base de datos y aplicaciones, procesando peticiones antes de que lleguen al controlador final.

ORM (Object-Relational Mapping)
Técnica de programación para convertir datos entre el sistema de tipos utilizado en un lenguaje de programación orientado a objetos y la base de datos.
